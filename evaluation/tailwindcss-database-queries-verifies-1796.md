A repeatable agent workflow that verifies database queries in TailwindCSS and stops when no duplicate imports remain.

Analyze the TailwindCSS repository. Focus specifically on database queries. Run the local inspection checks, locate any anomalies, drift, or inefficiencies. ```

## 🏁 Implementation Steps
1. Scan the TailwindCSS codebase for active database queries.
2. Identify any deviations from the target standard: no duplicate imports remain.

4. Rerun the verification command to check the updated state.
5. Log output details and repeat until no deviations remain.
