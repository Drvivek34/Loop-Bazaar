## 📝 Description
A repeatable agent workflow that debugs access control lists in TailwindCSS and stops when performance benchmark is met.

## 🎯 Use Case (When to Use)
> Use this whenever you need to check or modify access control lists in TailwindCSS systems and need a strict exit criteria.

## ⚡ System Prompt / Instructions
```
Analyze the TailwindCSS repository. Focus specifically on access control lists. Run the local inspection checks, locate any anomalies, drift, or inefficiencies. Modify the relevant code paths, verify the modifications against the local checks, and repeat the cycle until the stopping condition is fully met: performance benchmark is met.
```

## 🏁 Implementation Steps
1. Scan the TailwindCSS codebase for active access control lists.
2. Identify any deviations from the target standard: performance benchmark is met.
3. Implement corrections, optimization scripts, or code repairs.
4. Rerun the verification command to check the updated state.
5. Log output details and repeat until no deviations remain.

## 🛑 Stopping Condition (Verification)
**Verification Check**: Access control lists meets target standards.
- *Detail*: Checked and confirmed that: performance benchmark is met.

This loop works because it creates a clear evidence trail of system changes and performance checks.
