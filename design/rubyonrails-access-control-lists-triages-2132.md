## 📝 Description
A repeatable agent workflow that triages access control lists in Ruby on Rails and stops when documentation is fully aligned with implementation.

## 🎯 Use Case (When to Use)
> Use this whenever you need to check or modify access control lists in Ruby on Rails systems and need a strict exit criteria.

```
Analyze the Ruby on Rails repository. Focus specifically on access control lists. Run the local inspection checks, locate any anomalies, drift, or inefficiencies. Modify the relevant code paths, verify the modifications against the local checks, and repeat the cycle until the stopping condition is fully met: documentation is fully aligned with implementation.
```

## 🏁 Implementation Steps
1. Scan the Ruby on Rails codebase for active access control lists.
2. Identify any deviations from the target standard: documentation is fully aligned with implementation.
3. Implement corrections, optimization scripts, or code repairs.
4. Rerun the verification command to check the updated state.
5. Log output details and repeat until no deviations remain.

- *Detail*: Checked and confirmed that: documentation is fully aligned with implementation.
